# 更新日志

## 2026-01-14

### 新增功能

#### 1. 任务失败时自动重试机制
- 当任务执行失败时，系统会自动尝试重新登录账户，然后重新执行失败的任务
- 适用于因会话过期或网络临时故障导致的任务失败
- 实现位置：`scheduler.py` 中的 `ScheduledTask.execute()` 方法

**工作流程：**
1. 任务首次执行失败
2. 系统自动调用 `login()` 重新登录
3. 如果登录成功，重新执行任务
4. 如果重试后仍失败，记录错误日志

**示例场景：**
- 会话过期导致的 API 请求失败
- 临时网络波动导致的连接失败

#### 2. 错误聚合邮件功能
- 连续发生的多个错误会被聚合到一条邮件中发送，避免邮件轰炸
- 默认聚合窗口为 5 分钟（可配置）
- 在聚合窗口内的所有错误会被收集，统一发送一封汇总邮件
- 实现位置：`logger.py` 中的 `Logger` 类

**工作流程：**
1. 错误发生时，被添加到待聚合列表
2. 系统检查是否达到发送条件（首次错误或距上次发送超过聚合窗口）
3. 达到条件时，将所有待聚合的错误打包成一封邮件发送
4. 清空待聚合列表，等待下一个聚合周期

**邮件格式：**
```
主题：【系统错误汇总】UESTCService - N 个错误

内容：
在过去的时间窗口内，系统累计发生了 N 个错误：

============================================================

错误 1:
  时间: 2026-01-14 10:30:15
  消息: 获取用电数据异常: Connection timeout

错误 2:
  时间: 2026-01-14 10:35:20
  消息: 请求成绩 API 失败: HTTP 500

============================================================

此为系统自动告警邮件，请勿回复。
```

### 技术细节

#### 修改的文件

1. **scheduler.py**
   - `ScheduledTask.__init__()`: 添加 `retry_callback` 参数
   - `ScheduledTask.execute()`: 实现失败重试逻辑
   - `Scheduler.__init__()`: 添加 `retry_callback` 参数
   - `Scheduler.add_task()`: 传递重试回调给任务

2. **service_system.py**
   - `UESTCServiceSystem.__init__()`: 初始化调度器时传入 `self.login` 作为重试回调

3. **logger.py**
   - `Logger.__init__()`: 添加错误聚合相关属性
     - `error_aggregate_window`: 聚合时间窗口（默认 300 秒）
     - `pending_errors`: 待发送错误列表
     - `last_error_send_time`: 上次发送时间
   - `Logger._should_send_aggregated_errors()`: 判断是否应该发送聚合邮件
   - `Logger._send_aggregated_errors()`: 发送聚合错误邮件
   - `Logger.error()`: 重构为添加错误到聚合列表而非立即发送
   - `Logger.flush_errors()`: 新增方法，用于强制发送所有待聚合错误

### 配置说明

错误聚合窗口可以在创建 Logger 时配置：

```python
logger = Logger(
    name="UESTCService",
    log_dir="log",
    error_aggregate_window=300  # 5 分钟，可根据需要调整
)
```

### 使用建议

1. **错误聚合窗口设置**
   - 如果系统错误频率较高，可以适当延长聚合窗口（如 600 秒）
   - 如果需要及时收到错误通知，可以缩短聚合窗口（如 180 秒）

2. **程序退出时**
   - 建议在程序退出前调用 `logger.flush_errors()` 确保所有错误都被发送

3. **重试机制**
   - 重试仅执行一次，避免无限循环
   - 如果任务持续失败，应检查网络连接或账户状态
